Prerequisites & Toolchain
PHASE 01
x86 registers, flags, memory model, NASM assembly basics, cross-compiler setup, QEMU, GDB. The foundation everything else sits on.
▶ Videos
Nanobyte — Building an OS Series
videofree
The primary series. Covers x86 architecture, GRUB, protected mode, GDT/IDT in order. Start here and follow it alongside building Keti OS.
youtube.com/@nanobyte-dev
Low Level Learning
videofree
Short practical videos on x86 assembly, registers, pointers, and GCC toolchain internals. Great when a concept from Nanobyte needs more explanation.
youtube.com/watch?v=6S5KRJv-7RU
Computerphile — Memory & CPU explainers
videofree
5–15 min conceptual videos by Dr. Steve Bagley. Watch 'How Computer Memory Works' and 'Context Switching' before implementing those features.
youtube.com/@Computerphile
📚 Books
PC Assembly Language — Paul Carter
bookfree
NASM x86 32-bit assembly from scratch. Exactly the subset you need — real mode, protected mode, calling conventions. Free and legal PDF.
pacman128.github.io/pcasm
OS: Bootstrap Yourself (os01)
bookfree
Teaches you to read Intel manuals yourself. Covers x86 assembly, ELF, linking, debugging from first principles. Rare long-term understanding builder.
github.com/tuhdo/os01
🌐 References
OSDev Wiki
reffree
Every GDT, IDT, PIC, PIT, paging detail is here. Not linear reading — use it like a reference manual when you get stuck on anything.
wiki.osdev.org
Intel x86 Software Developer's Manual
reffree
The official ground truth for every instruction, flag, and register. Use Vol. 1 (basics) and Vol. 3A (protection, paging) most often.
intel.com — SDM
cfenollosa/os-tutorial (GitHub)
reffree
30 progressive coding exercises. Real mode → protected mode → VGA → keyboard. Pure learning-by-doing. Perfect companion for Phases 1–3.
github.com/cfenollosa/os-tutorial
Booting
PHASE 02
Power-on → MBR → bootloader → kernel. You've already completed the bootloader step with your Keti OS boot message. Now it's GRUB and getting into C.
▶ Videos
Nanobyte Parts 1–4
videofree
FAT12 disk loading, GRUB multiboot, protected mode, jumping to C. Watch in order — this is the exact flow Keti OS will follow.
youtube.com/@nanobyte-dev
Poncho's OS Dev Series
videofree
Thorough older series covering real mode, A20 line, GDT, protected mode transition with live coding. Good when Nanobyte doesn't click.
youtube.com — Poncho OS playlist
📚 Books
Writing a Simple OS from Scratch — Nick Blundell
bookfree
~55 pages. Covers 16-bit boot sector, protected mode, C kernel entry. Read this before anything else in Phase 2 — the clearest starter text.
cs.bham.ac.uk — os-dev.pdf
The Little Book about OS Development
bookfree
Swedish university course book. GRUB → framebuffer → GDT → IDT → PMM → paging. One of the two core texts for this entire journey.
littleosbook.github.io
🌐 References
OSDev Wiki: Bare Bones Tutorial
reffree
Official GRUB + Multiboot minimal kernel walkthrough. Covers linker script, multiboot header, and cross-compiler in 20 min of reading.
wiki.osdev.org/Bare_Bones
GRUB Multiboot Specification
reffree
The official spec for the magic header GRUB checks. Short read — tells you exactly what your kernel needs for GRUB to load it.
gnu.org — multiboot spec
Kernel Foundations
PHASE 03
VGA output, GDT, IDT, interrupts, keyboard driver, timer. This is where your OS goes from 'it boots' to 'it does things'. Expect IDT wiring to be fiddly.
▶ Videos
Nanobyte Parts 5–9
videofree
GDT, IDT, PIC remapping, keyboard IRQ, timer. His PIC remapping and ISR stub videos are exceptionally clear.
youtube.com/@nanobyte-dev
James Molloy's Kernel Dev Tutorials
writtenfree
The classic written tutorial. VGA, GDT, IDT, IRQs, timer, keyboard — each in its own chapter with full C code. Has been the starting point for many hobby OSes.
jamesmolloy.co.uk/tutorial_html
🌐 References
OSDev Wiki: Interrupts, PIC, IDT
reffree
IDT structure, 8259 PIC remapping, ISR stubs. Bookmark all three pages — you will hit them constantly during this phase.
wiki.osdev.org/Interrupts
OSDev Wiki: PS/2 Keyboard
reffree
Scancode sets, port 0x60/0x64, IRQ1 handling. The definitive keyboard driver reference.
wiki.osdev.org/PS/2_Keyboard
Memory Management
PHASE 04
Physical memory manager, paging (virtual memory), kernel heap. The hardest conceptual phase. Take it slow — paging bugs cause the most cryptic crashes.
▶ Videos
Nanobyte — PMM & Paging episodes
videofree
Bitmap PMM over the Multiboot memory map. One of the clearest frame allocation explanations on YouTube.
youtube.com/@nanobyte-dev
Computerphile — Virtual Memory series
videofree
3-part series on why virtual memory exists, TLB, and page tables. Watch before writing any paging code or you'll cargo-cult it.
youtube.com/@Computerphile
MIT 6.004 — Memory Hierarchy (OCW)
videofree
Free MIT lectures. Deep understanding of caches and page tables. Use when you want to go beyond the tutorial level.
ocw.mit.edu — 6.004
📚 Books
OS: Three Easy Pieces — Part II (Memory)
bookfree
The best OS theory book available, free from the authors. Part II covers paging, TLBs, page tables, and swap deeply. Read the relevant chapter the same week you implement it.
pages.cs.wisc.edu/~remzi/OSTEP
The Little Book — Paging Chapter
bookfree
More practical than OSTEP — shows the exact CR3/CR0 sequence and page directory layout for x86. Keep open while coding.
littleosbook.github.io/#paging
🌐 References
OSDev Wiki: Paging
reffree
x86 page directory, page table entry bit layout, enabling paging step-by-step. The definitive quick reference for Phase 4.
wiki.osdev.org/Paging
Processes & Shell
PHASE 05
Scheduling, context switching, system calls, user mode, and a working shell. The payoff phase — this is where Keti OS starts to feel like a real OS.
▶ Videos
Computerphile — Context Switching
videofree
15 min with Dr. Steve Bagley. Watch before implementing your scheduler — the clearest explanation of what actually happens during a task switch.
youtube.com/watch?v=DKmBRl8j3Ak
MIT 6.S081 — OS Engineering (full free course)
videofree
MIT's actual undergraduate OS course using xv6. All lectures and labs freely available. Covers syscalls, process management, scheduling, and the shell in great depth.
pdos.csail.mit.edu/6.S081
📚 Books
xv6 — MIT Teaching OS + Commentary Book
bookfree
A full Unix-like OS in ~6000 lines of readable C. The companion book explains every subsystem line by line: scheduler, syscalls, shell, filesystem. Your north star for Phase 5–6.
pdos.csail.mit.edu/6.1810/xv6.html
OS: Three Easy Pieces — Part I (CPU)
bookfree
Process abstraction, limited direct execution, scheduling policies, and context switching — all explained conceptually before you implement them.
pages.cs.wisc.edu/~remzi/OSTEP
🌐 References
OSDev Wiki: Scheduling & Context Switching
reffree
TSS, ring 3 entry, the iret trick for first task switch. Critical reading before implementing.
wiki.osdev.org/Scheduling_Algorithms
OSDev Wiki: System Calls
reffree
int 0x80 dispatch, argument passing. Start with int 0x80 — simpler. Implement write(), read(), exit() first.
wiki.osdev.org/System_Calls
Filesystem & ELF Loading
PHASE 06
VFS abstraction, ramdisk, and ELF binary loading. This is what lets you run chess as a real process inside Keti OS.
▶ Videos
MIT 6.S081 — Filesystem Lectures (14–16)
videofree
Inode-based filesystems, buffer cache, and logging. Dense but the clearest university-level treatment available free.
pdos.csail.mit.edu/6.S081
📚 Books
OS: Three Easy Pieces — Part III (Persistence)
bookfree
File systems, I/O devices, journaling. The FS chapters are excellent when you tackle Phase 6.
pages.cs.wisc.edu/~remzi/OSTEP
xv6 Book — Filesystem Chapter
bookfree
Walking through a real FS implementation — logging, inodes, directories — in readable C. Much more useful than theory alone.
pdos.csail.mit.edu/6.1810/xv6.html
🌐 References
OSDev Wiki: Initrd / Ramdisk
reffree
Embed a tar in the kernel image, parse it at boot. Gives you a read-only filesystem without writing a disk driver.
wiki.osdev.org/Initrd
OSDev Wiki: ELF
reffree
ELF header parsing, PT_LOAD segments, entry point. You need this to exec() an external binary inside Keti OS.
wiki.osdev.org/ELF
System V ELF ABI Specification
reffree
The official ELF format spec. Use when the wiki isn't specific enough about a field or segment type.
refspecs.linuxbase.org/elf/elf.pdf
